geocodificacion_convencional


La geocodificación convencional es el proceso de localización de una dirección junto con su ubicación (coordenadas geográficas).

El caso más simple es la localización indicando una cadena de texto que forme parte de la dirección.


Construcción de la consulta

El formato de interrogación a la API se conforma a partir de la url para el método elegido (https://geocoder-5-ign.larioja.org/v1/search), seguido del signo ? y de la secuencia de parámetros que configuran la pregunta.

El parámetro a utilizar es:

Parámetro significado Tipo Requerido Defecto Ejemplo
text texto de la dirección a buscar string yes none Calle Mayor

En el ejemplo siguiente (1) el texto a buscar en la dirección es «murrieta», para lo cual se añade a la url el parámetro text=murrieta.
https://geocoder-5-ign.larioja.org/v1/search?text=murrieta.

Pulsando sobre el enlace anterior se obtiene un fichero en formato GoeJSON que contiene los mejores resultados de la búsqueda que coinciden con el texto murrieta. El formato JSON puede ser leido en un navegador web (1) o en un procesador de texto.

Los resultados obtenidos se corresponden con las siguientes direcciones: BARRIO MURRIETA, Putxeta, Bizkaia, España (-3.091738,43.319819)
CALLE MURRIETA, San Vicente de Barakaldo/San Bizenti-Barakaldo, Bizkaia, España (-2.989928,43.301062)
Calle Marqués de Murrieta, Logroño, La Rioja, España (-2.452761,42.465614)


Determinar el nùmero de resultados

Por defecto la API devuelve hasta 10 resultados. Si se desea un número distinto de resultados, lo puede configurar en la llamada con el parámetro size.

Parámetro significado Tipo Requerido Defecto Ejemplo
size número deseado de resultados integer no 10 5


(1) Se aconseja implementar en el navegador web un complemento para la visualización estructurada de ficheros JSON..
